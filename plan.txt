Plan: Contrastive Self-Supervised Learning for Cold-Start Drug–Target Affinity Prediction (CL-DTA)
TL;DR — Cold-start generalization (predicting affinity for entirely unseen drugs or targets) is the most realistic yet most challenging DTA scenario. Existing models train purely supervised and suffer large performance drops under cold splits. We propose CL-DTA: a two-phase framework that first pretrains drug and protein encoders via augmentation-based contrastive learning (SMILES enumeration, subsequence cropping, atom/residue masking), then fine-tunes with supervised affinity regression. The key hypothesis is that contrastive pretraining forces encoders to learn augmentation-invariant, structurally meaningful representations that transfer better to unseen entities. We evaluate on DAVIS and KIBA under four split protocols (random, cold-drug, cold-target, cold-both) against five baselines, with ablations on augmentation strategy, loss function, and pretraining scale. The project builds directly on your existing DeepDTA CNN pipeline.

Phase 1 — Contrastive Pretraining Module (Weeks 3–6)

Create augmentations.py with the following drug & protein augmentation strategies:

Drug (SMILES) augmentations:
SMILES enumeration: use RDKit to generate non-canonical SMILES (same molecule, different string) — the strongest augmentation
Atom masking: randomly replace k% of SMILES tokens with a <MASK> token
Substructure dropout: randomly remove single atoms/bonds from the SMILES string (validated via RDKit to ensure valid molecule)
Protein (sequence) augmentations:
Subsequence cropping: randomly sample a contiguous window of 70–100% of the sequence length
Residue masking: randomly replace k% of amino acid characters with <MASK>
Residue substitution: replace residues with biochemically similar amino acids (using BLOSUM62 substitution probabilities)
Each augmentation returns a new string; the tokenizer in tokenizers_and_datasets.py handles conversion to tensors.

Phase 7 — Reproducibility & Code Release (Weeks 18–22)

Final code structure:

Verification
Correctness: Unit tests for augmentations (augmented SMILES produce valid molecules), metrics (match scipy), and data splits (no leakage verified by checking drug/target ID overlap between train/test)
Reproducibility: Fixed seeds (3 runs), pinned dependencies, saved configs per experiment, checkpoint saving
Statistical rigor: 5-fold CV × 3 seeds, paired significance tests (p < 0.05), confidence intervals reported
Sanity checks: random split performance should match published DeepDTA numbers (CI ≈ 0.878 on DAVIS); if not, debug before proceeding
Smoke test command: python scripts/run_finetune.py --config configs/davis_cold_drug.yaml --epochs 5 should run without errors and produce metrics
Decisions
Contrastive Cold-Start DTA chosen over uncertainty-aware, cross-attention, and graph-hybrid directions — strongest literature gap, builds cleanly on existing cold-split code
CNN encoders retained (not Transformers) — feasible on consumer GPU, sufficient for proving the contrastive pretraining hypothesis, Transformer extension is future work
NT-Xent as primary loss — best established contrastive loss, InfoNCE as ablation alternative
DAVIS + KIBA as dual benchmarks — standard in DTA literature, sufficient for IEEE journal
4 split protocols including cold-both — differentiator vs most DTA papers that only test random splits
No external pretraining corpus in v1 — pretrain on DAVIS/KIBA training sets only, ChEMBL extension is future work (keeps scope manageable)
